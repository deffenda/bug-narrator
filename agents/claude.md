# Claude Execution Contract

- Read the canonical roadmap and execution state before proposing or executing changes.
- Reuse the existing roadmap, risk, and task ledgers instead of creating shadow planning files.
- Treat evidence as mandatory for code, workflow, or config changes, and record exact commands plus outcomes in `state/session.json`.
- When you discover a blocker or regression, keep the unresolved risk ID alive or add a new logged risk before ending the slice.
- Run the repo validator before concluding substantive work and respect its `FAIL` output as a blocking condition.
